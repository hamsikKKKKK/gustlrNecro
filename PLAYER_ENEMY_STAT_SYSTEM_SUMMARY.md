## 가장 큰 방향

이번 작업의 방향은 크게 4가지다.

1. 플레이어와 잔몹이 서로 다른 방식으로 숫자를 들고 있지 않게 만들기
2. 플레이어는 나중에 아이템, 레벨업, 버프/디버프가 붙어도 구조가 안 깨지게 만들기
3. 전투가 실제로 동작하게 만들기
4. 스포너가 너무 단순하게 계속 몬스터를 보충하지 않도록 웨이브 개념을 넣기

**스탯, 전투, 피격, 사망, 웨이브 리스폰의 최소 골격을 만든 상태**다.

## 현재 파일별 역할

### 1. `Assets/02.Scripts/Core/Stats/CharacterStats.cs`

이 파일은 **공통 스탯 런타임 계산기**다.

이 클래스는 "기본 스탯 값 그 자체"가 아니다.
정확히는 아래 3개를 같이 관리한다.

- base stat
- modifier
- final stat

즉 의미는 이렇다.

- base stat
  - 캐릭터 원래 기본값
  - 예: 이동속도 5, 체력 100, 공격력 10
- modifier
  - 외부에서 붙는 추가값
  - 예: 공격력 +5, 이동속도 +10%
- final stat
  - 실제 게임에서 쓰는 최종값

### 지금 지원하는 공통 스탯 종류

- `MoveSpeed`
- `MaxHealth`
- `AttackPower`

### 지금 지원하는 modifier 방식

- `Flat`
  - 예: 공격력 +5
- `PercentAdd`
  - 예: 공격력 +10%
- `PercentMultiply`
  - 예: 계산 완료 후 한 번 더 곱하는 최종 배율

### 이 클래스가 하는 일

- 기본값 저장
- modifier 저장
- 최종값 계산
- 현재 체력 저장
- 체력 감소 / 회복
- 이벤트 발행

### 이벤트

- `StatChanged`
- `HealthChanged`

이 이벤트들은 나중에 UI가 붙을 때 그대로 쓰면 된다.
즉 지금은 UI가 없지만, 나중에 체력바/스탯창이 생기면 이 이벤트를 구독하면 된다.

### 중요한 성능 포인트

이 클래스는 **매 프레임 계속 다시 계산하는 구조가 아니다.**

최종 스탯 계산은 아래처럼 **값이 바뀌는 순간에만** 다시 돌도록 되어 있다.

- base stat이 바뀔 때
- modifier가 추가될 때
- modifier가 제거될 때
- modifier 전체를 비울 때

이동이나 공격 중에는 이미 계산된 최종값을 읽기만 한다.

즉:

- 변화 없음 → 재계산 안 함
- 아이템/레벨업/버프 변화 있음 → 그때만 재계산

### 2. `Assets/02.Scripts/Player/PlayerController.cs`

이 파일은 **플레이어 본체 컨트롤러**다.

### 담당 역할

- 플레이어 싱글톤
- 입력 처리
- 이동 처리
- 방향 처리
- 애니메이션 상태 처리
- 위치 높이 보정
- 플레이어 사망 시 정지 처리

### 지금 들고 있는 기본 스탯

- `baseMoveSpeed`
- `baseMaxHealth`
- `baseAttackPower`

중요한 점은:

- `PlayerController`가 최종 스탯 계산을 직접 하지 않는다.
- 계산은 `PlayerStats`와 `CharacterStats`가 한다.
- `PlayerController`는 그 결과를 읽어서 이동 속도 등에 사용한다.

즉 역할 분리가 되어 있다.

- `PlayerController` = 조작 / 움직임 / 방향 / 사망 제어
- `PlayerStats` = 플레이어 수치 제어

### 현재 이동 입력

현재 이동은 **WASD만 사용**한다.

- `W` = 위
- `A` = 왼쪽
- `S` = 아래
- `D` = 오른쪽

방향키는 공격용으로 분리했다.

즉 현재 입력은 아래처럼 나뉜다.

- `WASD` = 이동
- `방향키` = 공격

### 현재 플레이어 사망 처리

플레이어 HP가 0이 되면:

- 이동 정지
- 공격 모듈 비활성화
- 컨트롤러 비활성화
- 콘솔 로그 출력

현재는 UI 없이 로그만 뜬다.

예상 로그:

- 피해를 받았을 때
  - `[Player] 피해 X 받음 | HP 현재/최대`
- 회복했을 때
  - `[Player] 회복 X | HP 현재/최대`
- 죽었을 때
  - `[Player] HP가 0이 되어 사망했습니다.`

### 3. `Assets/02.Scripts/Player/PlayerStats.cs`

이 파일은 **플레이어 전용 스탯 진입점**이다.

플레이어는 앞으로 다음 시스템이 붙을 가능성이 높다.

- 아이템
- 레벨업
- 버프/디버프
- UI

그래서 공통 스탯 계산기 `CharacterStats`를 바로 쓰지 않고,
그 앞에 플레이어 전용 창구를 하나 둔 것이다.

### 역할

- 플레이어 기본 스탯을 `CharacterStats`에 넣기
- 플레이어 피격 / 회복 처리
- modifier 적용
- modifier 제거
- UI가 보기 좋은 플레이어 전용 접근점 제공

### 핵심 메서드

- `ConfigureBaseStats(...)`
  - 플레이어 기본 스탯 설정
- `TakeDamage(...)`
  - 플레이어 HP 감소
- `Heal(...)`
  - 플레이어 HP 회복
- `ApplyModifier(...)`
  - modifier 1개 적용
- `ApplyModifiers(...)`
  - 여러 modifier 적용
- `ApplyOrReplaceSourceModifiers(...)`
  - 같은 source가 넣은 modifier 교체
- `RemoveModifiersFromSource(...)`
  - source 기준 제거

### 왜 source가 중요한가

modifier는 나중에 아래처럼 들어올 가능성이 있다.

- 아이템 인스턴스
- 레벨업 시스템
- 스킬 버프
- 디버프 상태

예를 들어 "검 A가 공격력 +5를 줬다"면,
나중에 검 A를 해제할 때 검 A가 넣었던 값만 정확히 제거할 수 있어야 한다.

그래서 modifier에는 source 개념이 들어가 있다.

### 4. `Assets/02.Scripts/Player/PlayerAttackModule.cs`

이 파일은 **플레이어 기본 공격 모듈**이다.

이전에는 플레이어가 공격력 수치만 있고 실제 공격 로직은 없었다.
지금은 이 모듈이 플레이어 공격을 담당한다.

### 현재 공격 입력

- 방향키 위 = 위 공격
- 방향키 아래 = 아래 공격
- 방향키 왼쪽 = 왼쪽 공격
- 방향키 오른쪽 = 오른쪽 공격

즉 플레이어는 현재:

- `WASD`로 움직이고
- `방향키`로 공격한다

### 현재 공격 방식

처음에는 전방 박스 판정이었다.
하지만 화면상으로 "분명 앞인데 안 맞는" 느낌이 강해서
지금은 **전방 부채꼴 판정**으로 바꿨다.

### 현재 공격 판정 구조

공격할 때 내부적으로는 이렇게 처리한다.

1. 방향키 입력으로 공격 방향을 정한다
2. 플레이어가 그 방향을 바라보게 만든다
3. 플레이어 앞쪽에 원형 범위로 후보를 찾는다
4. 후보들 중에서 전방 각도 안에 들어온 적만 남긴다
5. 그 적들에게 데미지를 준다
6. 맞은 적은 약하게 넉백한다
7. 로그를 출력한다

### 현재 공격 파라미터

- `attackCooldown`
  - 공격 쿨타임
- `attackOriginOffset`
  - 판정 시작 위치를 몸 앞쪽으로 얼마나 밀지
- `attackRadius`
  - 부채꼴 판정 반경
- `attackAngle`
  - 부채꼴 각도
- `attackHeightOffset`
  - 판정을 띄우는 높이
- `enemyKnockbackDistance`
  - 적이 맞았을 때 밀리는 거리

### 적중 로그

적을 맞췄을 때:

- `[PlayerAttack] 방향 공격 적중 | 대상: 이름 | 데미지: 값`

아무도 안 맞았을 때:

- `[PlayerAttack] 방향 공격 | 데미지: 값 | 적중 대상 없음`

### 현재 공격 데미지 기준

플레이어가 실제로 주는 데미지는 `PlayerController.AttackPower`를 쓴다.
즉 최종적으로는 `PlayerStats` -> `CharacterStats` 에서 계산된 값을 사용한다.

즉 나중에 아이템/레벨업으로 공격력이 오르면 이 공격 모듈도 자동으로 강해진다.

### 5. `Assets/02.Scripts/Enemy/EnemyController.cs`

이 파일은 **적 AI + 적 스탯 + 적 전투 처리 중심**이다.

### 담당 역할

- AI 상태 전환
  - Idle
  - Wander
  - Chase
  - Attack
  - Dead
- 이동
- 플레이어 추적
- 플레이어 공격
- 피격 처리
- 넉백 처리
- 풀링 연동

### 적 스탯 사용 방식

적도 플레이어처럼 `CharacterStats`를 내부에 가진다.

즉 적도 더 이상 아래 숫자를 별도 변수로 직접 관리하지 않는다.

- 이동속도
- 체력
- 공격력

적 스폰 시 `EnemySpawnRuleConfig` 값을 읽어서 `CharacterStats`에 넣고,
그 뒤엔 최종값을 읽어서 동작한다.

예:

- 이동할 때 `stats.MoveSpeed`
- 공격할 때 `stats.AttackPower`
- 죽었는지 볼 때 `stats.IsDead`

### 적의 플레이어 공격

적이 공격 범위에 들어오면 `TryPerformAttack(...)`에서 플레이어에게 데미지를 준다.

현재는:

- 적 공격력만큼 플레이어 HP 감소
- 콘솔 로그 출력
- 플레이어 HP가 0이면 플레이어 사망 처리

즉 지금은 UI 없이도 전투 테스트가 가능한 상태다.

### 적 피격 처리

플레이어 공격이 적에게 맞으면:

1. `enemy.TakeDamage(damage)` 호출
2. 적 HP 감소
3. HP가 0이면 Dead 상태 전환
4. 죽은 뒤 풀로 반환

### 적 넉백 처리

플레이어 공격에 맞은 적은 `ApplyKnockback(...)`로 아주 약하게 밀린다.

현재 넉백은:

- 공격 방향 기준으로 밀기
- 이동 가능한 지형일 때만 실제 위치 이동
- 밀린 뒤 기존 목적지 이동은 끊기게 처리

즉 "아예 튕겨 나가는 느낌"보다는
**맞았다는 반응이 보이는 정도의 약한 밀림**이다.

### 7. `Assets/02.Scripts/Enemy/EnemySpawner.cs`

이 파일은 **적 스포너 로직**이다.

이번 작업에서 가장 크게 바뀐 파일 중 하나다.

예전 방식은 사실상:

- 플레이어 근처면 적 생성
- 죽으면 빈 자리 보충

에 가까웠다.

지금은 이걸 **웨이브 + 카메라 밖 보충 규칙**으로 바꿨다.

### 현재 스포너 동작 핵심

#### 1. `maxAlive`는 이제 "한 웨이브 최대 수"

즉 스포너 하나가 한 번에 가질 수 있는 최고 잔몹 수다.

예:

- `maxAlive = 3`
  - 그 웨이브에서는 최대 3마리까지 생성

#### 2. 죽자마자 바로 보충하지 않음

현재는 적이 한 마리 죽었다고 바로 화면 안에서 1마리를 다시 뽑지 않는다.

대신 아래 조건이 맞을 때만 부족한 수를 채운다.

- 현재 살아있는 수가 `maxAlive`보다 적을 것
- `respawnCooldown`이 지났을 것
- 그 스포너가 현재 카메라 화면 밖에 있을 것

즉 전멸하지 않았더라도 일부가 죽어서 빈 자리가 생기면,
**화면 밖에서만 `maxAlive`까지 다시 보충하는 구조**다.

#### 3. 플레이어가 스포너 반경 밖으로 나가면

- 현재 살아있는 적은 풀로 회수
- 스포너 상태 초기화
- 다시 들어오면 새 웨이브부터 시작

즉 스포너는 플레이어 근처일 때만 의미 있게 동작한다.

### 현재 카메라 밖 판정 방식

스포너의 `anchorPosition`을 현재 카메라 viewport 좌표로 바꿔서,
화면 안에 있으면 리스폰하지 않고,
화면 밖일 때만 리스폰한다.

즉 플레이어가 지금 보는 근처에서 몬스터가 갑자기 보충되는 문제를 막는 구조다.

## 현재 플레이 흐름

## 1. 게임 시작

1. 플레이어 생성
2. `PlayerController`가 기본 스탯을 가짐
3. `PlayerStats`가 그 값을 `CharacterStats`에 넣음
4. 공격 모듈이 자동으로 붙음
5. 적 스포너는 플레이어 거리 기준으로 동작 준비

---

## 2. 플레이어 이동

- `WASD` 입력을 읽는다
- 플레이어 방향을 갱신한다
- 그 방향에 맞는 이동 애니메이션을 재생한다
- 실제 이동 속도는 `MoveSpeed` 최종값을 사용한다

즉 나중에 이동속도 modifier가 붙어도 이동 코드는 건드릴 필요가 없다.

---

## 3. 플레이어 공격

- 방향키를 누르면 공격 방향 결정
- 플레이어가 그 방향을 바라봄
- 부채꼴 판정 생성
- 범위 안 적을 찾음
- 각도 안에 들어온 적만 적중 처리
- 적은 데미지를 입고 약간 넉백
- 콘솔에 공격 로그 출력

---

## 4. 잔몹이 플레이어를 공격

- 적 FSM이 Attack 상태 진입
- 공격 쿨타임 확인
- 쿨타임이 끝났으면 플레이어에게 데미지 적용
- 플레이어 HP 감소 로그 출력
- HP가 0이면 플레이어 사망

---

## 5. 잔몹이 죽었을 때

- 적 HP가 0이 됨
- Dead 상태로 전환
- 풀로 반환
- 스포너는 현재 살아있는 수가 줄었다고 판단
- 이후 카메라 밖 + 쿨다운 완료 시 부족한 수를 다시 채움

---

## 6. 다음 웨이브 리스폰

죽었다고 바로 안 생긴다.

다음 3가지가 모두 맞아야 한다.

1. 현재 살아있는 수가 `maxAlive`보다 적음
2. 리스폰 쿨다운 완료
3. 카메라 화면 바깥

그때만 스포너가 부족한 수를 다시 채워서 `maxAlive`까지 맞춘다.

## 이번 작업의 핵심 장점

### 1. 스탯 구조가 분리됨

이제 플레이어와 적이 제각각 숫자를 들고 움직이지 않는다.
공통 계산 구조가 생겼다.

### 2. 플레이어 확장성이 생김

나중에 아래를 붙이기 쉬워졌다.

- 아이템
- 레벨업
- 버프/디버프
- UI

### 3. 전투가 실제로 됨

이전에는 공격력 숫자만 있고 실제 플레이어 공격 로직이 없었다.
지금은 플레이어 공격, 적 피격, 넉백, 플레이어 피격, 사망까지 이어진다.

### 4. 스포너가 덜 어색해짐

이전보다 "죽자마자 화면 안에서 즉시 보충되는" 느낌이 줄었다.
이제는 화면 밖 + 쿨다운 이후에만 부족한 수를 채우기 때문에 훨씬 자연스럽다.

## 현재 한계 / 아직 안 한 것

이번 작업에서 일부러 아직 안 붙인 것도 있다.

### 1. UI

아직 UI는 없다.

- 체력바 없음
- 공격 쿨다운 UI 없음
- 적 체력바 없음
- 스탯창 없음

대신 로그로 확인하는 상태다.

### 2. 아이템 시스템

구조는 아이템을 받을 준비가 되어 있지만,
실제 인벤토리/장착/드랍 시스템은 아직 안 만들었다.

### 3. 플레이어 공격 애니메이션

현재는 실제 판정/데미지는 있지만
"공격 전용 스프라이트 애니메이션"은 따로 없다.

즉 지금은 방향 전환 + 판정 + 로그 중심이다.

### 4. 플레이어 사망 후 리스폰 / 게임오버 흐름

지금은 HP가 0이 되면 죽고 멈춘다.
하지만 아직 아래는 없다.

- 게임오버 UI
- 허브로 복귀
- 체크포인트 리스폰
- 재시작 버튼

### 5. 적 공격 피격 반응

지금은 플레이어가 HP가 깎이고 죽는 것까지는 되지만,
플레이어가 적에게 맞았을 때의 별도 피격 애니메이션이나 경직은 없다.

## 앞으로 확장할 때 어디를 건드리면 되는가

### 아이템 추가

건드릴 곳:

- `PlayerStats`
- `CharacterStats`
- 아이템 데이터 구조

방식:

- 아이템이 modifier 목록을 들고 있게 만들기
- 장착 시 `ApplyOrReplaceSourceModifiers(...)`
- 해제 시 `RemoveModifiersFromSource(...)`

### 레벨업 추가

건드릴 곳:

- `PlayerStats`
- 레벨업 시스템

방식:

- 레벨업 보너스를 modifier로 넣거나
- 영구 base stat 재설정으로 넣기

### UI 추가

건드릴 곳:

- `PlayerStats`의 이벤트
- `CharacterStats`의 `HealthChanged`, `StatChanged`

### 무기 종류 추가

건드릴 곳:

- `PlayerAttackModule`

가능한 방향:

- 전방 부채꼴 유지
- 투사체형 무기
- 범위형 무기
- 공격 방향별 다른 판정

### 잔몹 종류 / 패턴 추가

건드릴 곳:

- `EnemySpawnRuleConfig`
- `EnemyController`
- FSM 상태

## 가장 중요한 개념만 다시 짧게 요약

### 스탯

- `CharacterStats` = 공통 스탯 계산기
- `PlayerStats` = 플레이어 전용 스탯 창구
- `PlayerController` = 플레이어 조작 담당

### 전투

- 플레이어는 방향키로 공격
- 공격은 전방 부채꼴 판정
- 적은 맞으면 데미지 + 약한 넉백
- 적은 플레이어를 때릴 수 있음
- 플레이어 HP는 실제로 줄어듦
- HP 0이면 플레이어 사망

### 스포너

- `maxAlive` = 스포너 1개가 한 번에 유지할 수 있는 최고 수
- 일부가 죽어도 즉시 보충하지 않음
- 카메라 밖 + 쿨다운 완료일 때만 부족한 수를 리스폰
