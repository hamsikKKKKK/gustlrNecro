# Necrocis 맵 시스템 기획서

## 개요

- **장르**: 로그라이크
- **컨셉**: 인체 장기 모티브
- **스타일**: 돈스타브 스타일 오픈필드
- **구조**: 중간방(허브) + 4개 독립 바이옴 맵

---

## 월드 구조

```
중간방 (허브)
   │
   ├─ 장 포털 → 장 오픈월드 (120×120)
   ├─ 간 포털 → 간 오픈월드 (150×150)
   ├─ 위 포털 → 위 오픈월드 (90×90)
   └─ 폐 포털 → 폐 오픈월드 (135×135)
```

---

## 중간방 (허브)

### 기본 정보
- **생성 방식**: 고정 맵 (절차적 생성 아님)
- **구성 요소**:
  - 재단 (최종 보스 진입용)
  - 4개 바이옴 포털

### 재단
- 중간보스 부산물(목) 4개 제물 → 대뇌 맵 이동

---

## 바이옴 구성

### 기본 정보

| 바이옴 | 모티브 | 크기 | 취약 클래스 | 디버프 |
|--------|--------|------|-------------|--------|
| 장 | 숲, 습지 | 120×120 | 없음 | 없음 |
| 간 | 늪, 강 | 150×150 | 마법사 | 방어력 10%↓ |
| 위 | 화산 | 90×90 | 궁수 | 방어력 5%↓ |
| 폐 | 초원, 안개 | 135×135 | 전사 | 피격시 넉백 |

### 바이옴별 환경 요소

**장 (Intestine)**
- 바닥재: 진흙, 이끼
- 점액 웅덩이: 이동속도 20%↓, 공격속도 10%↓
- 기생충: 체력 3, 공격력 0.3
- 경계: 점액 늪

**간 (Liver)**
- 바닥재: 잔디, 모래
- 독성 연못, 담즙 강, 마른 나무, 갈라진 땅
- 유해가스: 초당 0.8 HP 손실
- 경계: 독성 강

**위 (Stomach)**
- 바닥재: 바위(대리석), 용암
- 피 분수: 감지 5유닛, 10초 주기, 기생충 스폰 (HP 4, 공격력 0.5)
- 위액 웅덩이: 초당 1.0 HP 손실, 이탈 후 2초간 0.5 HP
- 경계: 위산 바다

**폐 (Lung)**
- 바닥재: 모래, 사바나
- 바이러스 부유물: 초당 0.5 HP, 랜덤 순찰, 지속 20초
- 가래 투사체: 감지 8유닛, 사거리 7유닛, 공격력 1.0, 딜레이 5초
- 모래바람: 20~40초 간격, 5초간 시야 30%↓
- 경계: 유독 안개

### 몬스터 배치
- 일반몹: 바이옴당 5~15마리
- 엘리트: 바이옴당 1~2마리
- 중간보스: 바이옴당 1마리

## 게임 흐름

### 진행 방식

1. **시작**: 중간방에서 시작
2. **바이옴 선택**: 4개 포털 중 선택하여 진입
3. **바이옴 탐험**: 해당 바이옴 오픈월드 탐험
4. **보스 처치**: 중간보스 처치 → 귀환 포털 생성 + 부산물 획득
5. **귀환**: 중간방으로 복귀
6. **반복**: 다른 바이옴 진입
7. **최종**: 재단에 부산물 4개 바침 → 대뇌 맵 이동

### 바이옴 재진입

| 항목 | 첫 진입 | 재진입 |
|------|---------|--------|
| 맵 | 새로 생성 | 새로 생성 |
| 몬스터 | 기본 스탯 | 강화 |
| 중간보스 | 고정 | 랜덤 재생성 |

### 최종 보스
- 척추 복도 → 대뇌(내장) 보스룸
- 보스 처치 후 X자 힌트 → 히든보스 맵 이동

## 맵 생성 파이프라인

### 중간방 (허브)
```
고정 맵 로드
   ↓
포털 4개 배치
   ↓
재단 배치
```

### 바이옴 맵 (진입 시 생성)
```
1. 지형 생성 (Noise)
       ↓
2. 맵 경계 생성 (바이옴별 테마)
       ↓
3. 타일맵 렌더링 (비트마스킹)
       ↓
4. 오브젝트 배치 (Poisson Disk)
       ↓
5. 보스 서식지 배치
       ↓
6. 몬스터 스폰
       ↓
7. 입구 포털 배치
```

## 핵심 기술

### 1. FastNoise Lite

오픈소스 노이즈 라이브러리. C# 지원, Unity 호환.

**지원 노이즈**

| 종류 | 특징 | 용도 |
|------|------|------|
| Perlin | 부드러운 연속 노이즈 | 고도 |
| Simplex | Perlin 개선판 | 지형 변화 |
| Value | 단순 보간 | 빠른 연산 |

**Fractal 옵션**

| 타입 | 설명 |
|------|------|
| FBm | 기본 프랙탈, 자연 지형 |
| Ridged | 강, 웅덩이 배치 |

**주요 파라미터**
- Frequency: 패턴 밀도
- Octaves: 레이어 수
- Seed: 랜덤 시드

**Necrocis 적용**
- 지형 변화: FBm + Simplex
- 웅덩이/강 배치: Ridged

---

### 2. Poisson Disk Sampling

최소 거리 유지하면서 랜덤 배치. 균일하면서 자연스러운 분포 생성.

**알고리즘 (Bridson's Algorithm)**
1. 첫 점 랜덤 배치
2. 주변에 후보점 생성
3. 최소거리 이상인 후보만 채택
4. 반복

**바이옴별 밀도**

| 바이옴 | radius | 밀도 |
|--------|--------|------|
| 장 (숲) | 2~3 | 빽빽함 |
| 간 (늪) | 3~4 | 중간 |
| 위 (화산) | 4~5 | 약간 듬성 |
| 폐 (초원) | 5~6 | 듬성듬성 |

**다중 레이어 배치**
- 1차: 큰 오브젝트 r = 4
- 2차: 중간 오브젝트 r = 2
- 3차: 작은 오브젝트 r = 1

---

### 3. 비트마스킹 AutoTile

주변 8칸 상태를 비트로 표현하여 타일 자동 선택.

**비트 배치**
```
[128][  1][  2]
[ 64][타일][ 4]
[ 32][ 16][  8]
```

**타일맵 레이어 순서 (아래 → 위)**
1. Cliff (절벽)
2. Floor (바닥)
3. Object (오브젝트)
4. Shadow (그림자)
5. Wall (벽)

---

### 4. UnityEngine.Pool

Unity 2021+ 내장 오브젝트 풀링 API.

**풀링 대상**

| 대상 | 풀 크기 |
|------|---------|
| 일반 몹 | 20 |
| 엘리트 | 5 |
| 투사체 | 50 |
| 이펙트 | 50 |

---

### 5. ScriptableObject

데이터를 에셋으로 분리하여 관리.

**필요한 ScriptableObject**

| 이름 | 내용 |
|------|------|
| BiomeData | 크기, 디버프, 취약 클래스, 몬스터 정보 |
| BiomeTileset | 타일, 환경 오브젝트, Poisson 거리 |

## 준비물

### 아트 에셋 (바이옴당 1세트)

**타일**
- 바닥 기본 + 랜덤 변형 3~4개
- 벽 (비트마스킹용 약 16종)
- 절벽, 그림자
- 경계 타일

**환경 오브젝트**
- 장: 이끼 바위, 나무, 점액 웅덩이
- 간: 마른 나무, 갈라진 땅, 독버섯, 독성 연못
- 위: 바위, 뼈, 용암 분출구, 피 분수
- 폐: 풀, 갈대, 부유물 스포너

**공통**
- 중간방 맵
- 재단
- 포털 (입구/출구)
- 보스 서식지
- 몬스터 프리팹
